Personality trait analysis (PTA)

Project Overview

Base is an open chatting app, and I participated in the development of an algorithm to measure the user's soft skill based on the user's text message. At the time of participating in the project, the focus was on presenting possibilities due to the lack of data at the initial stage.

Timeline 2019
Role Building ML models
Tools Python

Introduction

Purpose

  • Recently, the understanding of natural language by a computer using Natural Language Processing(NLP) is one of the emerging issues
  • Amont them, through Sentiment Analysis, the computer has reached the point of analyzing the user's emotions
  • If so, wouldn't it be possible to measure the user's soft skills and traits such as the user's leadership, collaboration through the user's text message?
  • This project presented that possibility

    • Method

    Feature extraction
  • Because I focused on suggesting the possibility of measure, I selected Influential among several soft skills
  • To measure this skill, the reply time in individual chat rooms and group chat rooms were selected and extracted as a feature

  • (Left) Users who received a reply within a day were displayed by setting the x-axis to personal chat and the y-axis to group chat receive time, representing a total of 437 users
  • (Right) A total of 392 users received a reply within 6 hours

    • Because there was a limited number of users who sign up for and frequently use the app, all 437 user data in the first figure were used, and models were implemented to divide into small groups whether or not users are judged to be influential, rather than exact figures.


    Supervised Learning
  • Training data: Test data = 4:1
  • Number of group: 4
    Group0: receive a reply within 1hour & Group1: between 1hour and 3hours
    Group2: between 3hours and 5hous & Group3: more than 5hours

  • K-nearest neighbors classification
      • Objects are classified by a majority vote of neighbors and assigned to the most common class among k nearest neighbors
  • Decision tree
      • It learns a simple decision rule inferred from data characteristics to create and use a model that predicts the value of a target variable
  • Support Vector Machines (SVM)
      • A model that defines a decision boundary, that is a baseline for classification
    Unsupervised Learning
  • K-means clustering
      • Groups the given data into k clusters, and operates in a way that minimizes the variance of the difference between each cluster and distance
  • Hierarchical clustering
      • A clustering algorithm that groups similar clusters together until finally a single case is formed.

    Result

    Supervised Learning
  • Assumed that the user's influence was greater when the reply time was short
  • That is, the closer to the origin of the scatter graph, the greater the influence

  • K-nearest neighbors classification
      • Accuracy about: 97.5%
  • Decision tree
      • Accuracy about: 86.4%
  • SVM
      • Accuracy about: 99.2%
    Unsupervised Learning
  • K-means clustering
      • When divided into two clusters (Left), the assumed results are the same, but in three clusters (Right) the results are different from the expected result
  • Hierarchical clustering


      • We expected a result that the user's influence was greater as we went to the origin, but different results were derived when two clusters were set in the result of using hierarchical clustering

    Conclusion

  • Since it is a proposal stage, the graph was confirmed in two dimensions using only two features that are easy to confirm with the naked eye
  • Increasing the number of dimensions by increasing the number of features is expected to lead to better results
    • Accuracy
  • Using supervised learning, it was confirmed that up to 4 groups were well divided
  • In the case of the decision tree, group2 was not well distinguished, but it is necessary to check with more data because the number of data is small
  • In the case of unsupervised learning, there are cases where simple division into two is possible, but the result was large due to the insufficient amount of data
  • Based on the results, we propose a method to measure the skill score by first using supervised learning and increasing the number of features and groups
    • Limitation
  • Since the number of active users using the chatting app was small, the data was skewed to both ends, so it was difficult to subdivide it into many groups and score it
  • Verification using more data is needed
