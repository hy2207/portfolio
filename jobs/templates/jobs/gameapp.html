Tug Of Logic App

Project Overview

This project implements the 'Tug of Logic' board game which is proposed by Michael Picard so that it can be played on mobile. I participated in implementing his idea into a mobile application. This game is for players who have different opinions on a subject and main purpose is to persuade other players who have different ideas.

Introduction

What's Tug of Logic?


  • This Tug of Logic board game is a competitive game, and player can win by persuading other players with a different opinions
  • On the Main claim(MC) set by the referee, players share their opinions (Reason in Play; Rip) and vote for each Rip
  • At the end, players vote to see if their opinions have changed from before the debate and select a winner who influenced their choices
  • Due to the board game, multiple players share information at once and play, so it is implemented by connecting to the firebase realtime database
    • Data Modeling


  • Total 5 models are implemented
  • I focused on implementing the Main Claim, Bout and ReasonInPlay parts

    • Features

    Design


  • Tried to put the features necessary for the game on one page as much as possible
  • Allow different buttons between referee and player by distinguishing them when logging in
  • On the page showing the voting result, it was set to update the result in real time by connecting to firebase and displayed as a graph

    • How to play

  • (Left) After login, referee can set MC and see the prior MC as well
  • (Right) Player can check MC and vote on it
  • Result of straw poll page's design is similar with final poll result page which is mentioned in design part

    • After checking result of poll, the main game page is displayed


  • (Left) In this main page, there are player name and result of straw poll in top-right
  • Player can submit their Rip and referee will choose one for current Rip
  • Each bout, player can vote that this Rip is established or contested, also can write comment if they want
  • (Right) It's for referee, he has buttons which can choose Rip, reassign Rip, show comment/list, go next bout and end game

  • (Left) Referee can select current Rip and re-assign the Rip to one player among all player
  • (Right) When referee clicks the ‘voting ground’ button, voting dialog pops up and player can vote on sufficiency of ground
  • This result will be updated not only in this page, but also in main page

  • (Left) Can compare initial and final poll results
  • (Right) Providing all the rip during the play so players can take a look ground rip again and write down the comment what rip changed their mind on main claim
